=== FIRST THINGS TO DO THIS SESSION ===
Role rules live in the repo (agent-rules/contributor.md via AGENTS.md, and .contributor.json). Follow those as the source of truth.

When the user starts a session, or asks to start/set up/begin, run the `sc-start` command. It confirms their profile, gets them on the right workspace, pulls the latest, and starts the app — all in one step. Don't reproduce those steps by hand; just run `sc-start` and relay its output in plain English.

`sc-start` may ask the user something and wait for their answer. Handle their reply:
- If it lists several possible workspaces and asks which to use: once they name one, switch to that branch, then run `sc-start` again.
- If it says no workspace exists and asks whether to create one: only if they say yes, create the branch it named from main, then run `sc-start` again.
- If it says the contributor profile is missing or incomplete: help them set it up per the contributor rules, then run `sc-start` again.
- If it says they have unsaved changes: offer to save (commit to their own branch) first, then run `sc-start` again.

Never switch or create a branch unless the user has confirmed it.
=== END FIRST THINGS ===

=== HOW TO WORK ON THIS MACHINE ===

WHO YOU'RE HELPING
- The person is not a developer. Explain everything in plain English.
- Never assume they know git, terminals, or code. Don't show raw commands unless they ask.
- Before doing anything destructive or irreversible, stop and ask a simple yes/no question.

RUNNING THE APP — always use these helper commands, never inline shell:
- start-app        starts the dev server and prints the localhost URL to open in the browser on this machine
- wait-for-app     waits until the server is responding, then prints the URL
- sync-gamedata    refreshes the gamedata snapshot in the background (non-blocking)
Do not reconstruct these with raw nohup, curl loops, or & backgrounding.

SAVING WORK (git)
- Commit the user's work to their own branch regularly, and describe it to them as "saving your progress."
- Never commit to main. Never force-push or rewrite history.
- Explain each save in one plain sentence (what was saved and why).

WHEN SOMETHING BREAKS
- Don't retry blindly. Say plainly what went wrong, then point the user to [#support-channel].
